# Cerebro — Product PRD

## Overview

Cerebro is an AI-assisted IT support triage platform designed for Managed Service Providers (MSPs).

It acts as a centralized workspace that sits on top of existing PSA systems, primarily Autotask, and augments ticket management with automated analysis, contextual data aggregation, and structured decision support.

Instead of technicians manually gathering information across multiple tools, Cerebro automatically collects operational context, analyzes the situation using AI, and presents a structured diagnostic workflow that helps technicians move from ticket intake to resolution faster and with fewer errors.

Cerebro does not replace the PSA.  
It enhances the technician workflow on top of it.

---

## Product Goal

The goal of Cerebro is to reduce the time and cognitive effort required to triage and diagnose IT support tickets by:

- automatically gathering relevant operational context
- running AI-assisted analysis and validation
- presenting a structured decision workflow for technicians

Cerebro helps technicians understand issues faster and apply consistent troubleshooting processes across the organization.

---

## Core Value Proposition

When a support ticket arrives, Cerebro automatically performs the following steps:

1. Retrieve the ticket and metadata from Autotask.
2. Collect related context from connected systems.
3. Run an AI analysis pipeline.
4. Produce:
   - a diagnosis hypothesis
   - policy validation
   - a recommended troubleshooting playbook.
5. Present everything inside a unified triage workspace.

The technician remains in control and validates the recommendations before executing any action.

This creates a structured **human-in-the-loop triage workflow**.

---

## System Integrations

Cerebro integrates with several MSP operational systems to collect context and assist the technician.

### Autotask (Primary Integration)

Autotask is the primary system of record and currently the **only two-way integration**.

Capabilities include:

- retrieving tickets
- retrieving ticket metadata
- retrieving related entities (accounts, contacts, configuration items)
- updating ticket information
- adding notes or actions back to the ticket

Autotask remains the authoritative system for ticket lifecycle management.

---

### NinjaOne

NinjaOne provides operational device context.

Cerebro may retrieve:

- device status
- alerts
- monitoring data
- recent device activity

This integration is **read-only**.

---

### IT Glue

IT Glue provides documentation and configuration context.

Cerebro may retrieve:

- device documentation
- network documentation
- asset references
- configuration information

This integration is **read-only**.

---

### Email Ingestion

Incoming emails related to support requests may be ingested and associated with tickets to provide additional context.

This integration is also **read-only**.

---

## Core Workflow

Typical ticket flow:

1. A support ticket is created in Autotask.
2. Cerebro retrieves the ticket and associated metadata.
3. Cerebro gathers contextual information from connected systems.
4. An AI pipeline analyzes the issue.
5. Cerebro generates structured outputs including:
   - issue understanding
   - possible root causes
   - policy validation
   - recommended troubleshooting steps.
6. The technician reviews the analysis.
7. The technician executes or adjusts the recommended actions.

---

## User Interface Concept

Cerebro uses a **tri-pane workspace interface** optimized for operational triage.

The interface includes:

### Ticket Context Panel
Displays ticket data and contextual information collected from integrated systems.

### AI Analysis Panel
Shows diagnosis hypotheses, validation checks, and reasoning generated by the AI pipeline.

### Conversation and Action Panel
A chat-style interface where the technician interacts with the system, asks questions, and executes actions.

This layout keeps context, reasoning, and action within a single operational workspace.

---

## Design Principles

Cerebro follows several core principles:

- human-in-the-loop decision making
- context aggregation before diagnosis
- transparent AI reasoning
- integration with existing MSP workflows
- minimal disruption to technicians already using PSA systems
